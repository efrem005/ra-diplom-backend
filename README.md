# Backend для RA16-Diploma

Интернет-магазин обуви. Сервер предоставляет REST API для получения каталога товаров, категорий, оформления заказов.

## Запуск

```bash
# установить зависимости
npm install

# обычная версия
npm start

# версия с ошибками и задержками (для тестирования)
npm run flaky
```

#### Сервер запускается на `http://localhost:7070`
#### Сервер на "render.com" `https://ra-diplom-backend.onrender.com`

---

## API Reference

Базовый URL: `http://localhost:7070/api`

### 1. Хиты продаж

**GET** `/api/top-sales`

Возвращает список популярных товаров (хиты продаж).

**Ответ:** Массив объектов `Item`.

**Пример ответа:**
```json
[
  {
    "id": 20,
    "category": 13,
    "title": "Кроссовки как у Pharrell Williams",
    "images": [
      "https://example.com/photo1.jpg",
      "https://example.com/photo2.jpg"
    ],
    "sku": "1000000",
    "manufacturer": "Chanel",
    "color": "Черный",
    "material": "Ткань",
    "reason": "Прогулка",
    "season": "Лето",
    "heelSize": "3 см.",
    "price": 12000,
    "oldPrice": 14000,
    "sizes": [
      { "size": "10 US", "available": true },
      { "size": "15 US", "available": false }
    ]
  }
]
```

**Поля объекта Item:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | Уникальный идентификатор товара |
| `category` | number | ID категории |
| `title` | string | Название товара |
| `images` | string[] | Массив URL изображений |
| `sku` | string | Артикул |
| `manufacturer` | string | Производитель |
| `color` | string | Цвет |
| `material` | string | Материал |
| `reason` | string | Повод (назначение) |
| `season` | string | Сезон |
| `heelSize` | string | Размер каблука |
| `price` | number | Цена в рублях |
| `oldPrice` | number? | Старая цена (если есть скидка) |
| `sizes` | Size[] | Массив размеров |

**Поля объекта Size:**

| Поле | Тип | Описание |
|------|-----|----------|
| `size` | string | Размер (например "10 US") |
| `available` | boolean | Доступен ли для покупки |

### 2. Категории

**GET** `/api/categories`

Возвращает список категорий товаров.

**Ответ:** Массив объектов `Category`.

**Пример ответа:**
```json
[
  { "id": 12, "title": "Мужская обувь" },
  { "id": 13, "title": "Женская обувь" },
  { "id": 14, "title": "Обувь унисекс" },
  { "id": 15, "title": "Детская обувь" }
]
```

**Поля объекта Category:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | Уникальный идентификатор категории |
| `title` | string | Название категории |

---

### 3. Список товаров

**GET** `/api/items`

Возвращает список товаров с поддержкой пагинации, фильтрации по категории и поиска.

**Параметры запроса:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `categoryId` | number | Фильтр по категории (опционально) |
| `offset` | number | Количество пропущенных записей для пагинации (по умолчанию 0) |
| `q` | string | Поисковый запрос по названию и цвету (опционально) |

**Примеры запросов:**

- Все товары (первые 6): `GET /api/items`
- С пагинацией: `GET /api/items?offset=12`
- По категории: `GET /api/items?categoryId=13`
- Поиск: `GET /api/items?q=Кеды`
- Комбинированный: `GET /api/items?categoryId=13&q=Черный&offset=6`

**Ответ:** Массив объектов `Item` (максимум 6 за раз).

**Пример ответа:**
```json
[
  {
    "id": 35,
    "category": 14,
    "title": "Кеды классические",
    "images": ["https://example.com/sneakers.jpg"],
    "sku": "1000015",
    "manufacturer": "Vans",
    "color": "Черный",
    "material": "Ткань",
    "reason": "Спорт",
    "season": "Лето",
    "heelSize": "1 см.",
    "price": 700,
    "oldPrice": 1000,
    "sizes": [
      { "size": "10 US", "available": true },
      { "size": "14 US", "available": false }
    ]
  }
]
```

**Примечание:** Если сервер вернул меньше 6 элементов — это последняя страница.

### 4. Детали товара

**GET** `/api/items/:id`

Возвращает полную информацию о товаре по его ID.

**Параметры:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `id` | number | Идентификатор товара в URL |

**Пример запроса:** `GET /api/items/66`

**Ответ:** Объект `Item` (структура как в примере выше).

**Пример ответа:**
```json
{
  "id": 66,
  "category": 13,
  "title": "Босоножки 'MYER'",
  "images": [
    "https://example.com/sandals_myer.jpg",
    "https://example.com/sandals_myer_2.jpg"
  ],
  "sku": "1000046",
  "manufacturer": "PAUL ANDREW",
  "color": "Чёрный",
  "material": "Кожа",
  "reason": "Прогулка",
  "season": "Лето",
  "heelSize": "5 см.",
  "price": 34000,
  "sizes": [
    { "size": "18 US", "available": true },
    { "size": "20 US", "available": true }
  ]
}
```

**Коды ответа:**

| Код | Описание |
|-----|----------|
| 200 | Товар найден |
| 404 | Товар не найден |

### 5. Оформление заказа

**POST** `/api/order`

Оформляет заказ на покупку товаров.

**Тело запроса (JSON):**

| Поле | Тип | Описание |
|------|-----|----------|
| `owner.phone` | string | Телефон покупателя (формат: +7XXXXXXXXXX) |
| `owner.address` | string | Адрес доставки |
| `items` | OrderItem[] | Список заказываемых товаров |

**Поля объекта OrderItem:**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | number | ID товара |
| `price` | number | Фиксированная цена на момент добавления в корзину |
| `count` | number | Количество единиц товара |

**Пример запроса:**
```json
{
  "owner": {
    "phone": "+79991234567",
    "address": "г. Москва, ул. Примерная, д. 10, кв. 5"
  },
  "items": [
    { "id": 66, "price": 34000, "count": 1 },
    { "id": 65, "price": 7600, "count": 2 }
  ]
}
```

**Ответ:**

| Код | Описание |
|-----|----------|
| 200 | Заказ успешно оформлен (может вернуть пустое тело или `{}`) |
| 400 | Ошибка валидации данных |
| 500 | Внутренняя ошибка сервера |

**Пример успешного ответа:**
```json
{}
```

**Пример ошибки:**
```json
{
  "error": "Некорректные данные заказа"
}
```

## Поиск

Поиск работает по следующим правилам:
- **По названию** — частичное совпадение без учёта регистра (например, «жар» найдёт «Туфли Жар-птицы»)
- **По цвету** — точное совпадение без учёта регистра (например, «чёрный» найдёт товары с цветом «Черный»)

---

## Ошибки

Все ошибки возвращаются в формате JSON:

```json
{
  "error": "Описание ошибки"
}
```

При использовании `npm run flaky` сервер может случайным образом:
- Возвращать ошибки 500
- Отвечать с задержкой (до нескольких секунд)
- Возвращать пустые ответы

Это полезно для тестирования обработки ошибок и лоадеров на фронтенде.
